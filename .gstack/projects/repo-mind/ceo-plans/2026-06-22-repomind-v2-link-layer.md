## Vision

RepoMind wins when **humans and agents read the same `docs/` files** and the **graph reflects reality** without manual `related:` gardening. v0.1 shipped MCP + Confluence shell + drafts. v2 makes **links first-class**: parsed from markdown, indexed once, consumed by graph, reader, editor, MCP, and `check`.

### 12-month ideal

```
CURRENT (v0.1.2)              v2 (next 4–6 weeks)              12-MONTH IDEAL
docs/ + MCP + UI tree    →    Link index + UX polish      →    Team knowledge OS in git
Graph = related: only         Wikilinks + backlinks            Agent + human co-author
Manual frontmatter links      Move/rename safe                 Link health always green
Page info always visible      Collapsible / focus mode         Notion-feel, zero lock-in
```

## Architecture — Link Layer

```
docs/*.md
    │
    ▼
DocIndex.refresh()
    ├── frontmatter (type, status, related:)
    └── body scan → wikilinks[], markdown links
    │
    ▼
LinkIndex (new, in-memory per refresh)
    ├── edges: related | wikilink | markdown-href
    ├── backlinks: slug → [sources]
    └── broken: target missing
    │
    ├── explore_graph (MCP + UI)
    ├── GET /api/backlinks/:slug
    ├── doc-panel (inline wikilink render + backlinks)
    ├── editor ([[ autocomplete)
    └── check (broken link violations)
```

### Data flow shadow paths

| Path | nil/empty | upstream error |
|------|-----------|----------------|
| Wikilink `[[missing]]` | edge + broken_links | shown in check + publish warn |
| Move file | slug may change | block or offer cascade update |
| Graph depth limit | truncated flag (existing) | user sees "expand depth" |
