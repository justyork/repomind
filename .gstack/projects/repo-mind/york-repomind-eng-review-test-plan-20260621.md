## Key Interactions to Verify
- install-to-whoa [E2E]: `npx repo-mind init` → paste one-line MCP config →
  agent asks a project question → agent calls search_docs/get_doc → returns a
  correct, grounded answer. Must complete in <5 min on the demo repo.
- create_draft lifecycle [E2E]: agent calls create_draft → human reviews the
  branch/PR → merges → runs the returned `cleanupCommand` → worktree removed,
  branch gone. Verify the user's working tree/HEAD were never touched.
- concurrent create_draft [E2E]: two create_draft calls in parallel → in-process
  mutex serializes them → both succeed, no git worktree lock error, two distinct
  branches/worktrees.
- run-from-subdir [E2E]: MCP launched with cwd set to a subpackage → walk-up
  discovery finds `.project-knowledge/` at repo root → tools return correct
  results (not silent empty).
- A/B kill-switch [EVAL]: 5–10 diverse project questions × 2 runs (plain
  markdown+CLAUDE.md vs RepoMind MCP) on the seeded ~15-doc demo repo. Capture
  median tokens-in-context (from agent usage metadata) + total hallucination
  points (human, 3-point rubric, blind to run). Reproducible scripted harness
  in `ab-demo/`. This is the go/no-go gate — if RepoMind shows no visible gap,
  STOP (per design doc P2).

## Edge Cases
- Slug path traversal: `get_doc`/`create_draft` with slug `../x`, `a/../../b`,
  `..%2f..`, absolute path, empty string, `/`-containing → all rejected with
  zero filesystem access outside `.project-knowledge/`. (Security gate A1.)
- No `.project-knowledge/` found (walk-up exhausts) → every tool returns
  `{found:false}`/[] and check/init print "run `repo-mind init`".
- Non-git repo + create_draft → `ENOENT_REPO` typed error.
- search_docs empty/whitespace query → []. >20 matches → capped + sorted.
- explore_graph depth 0 or negative → clamped to 1 (root only). Cyclic
  `related:` → visited-set cuts, no infinite loop. `truncated=true` when
  frontier cut at requested depth.
- get_glossary_term: no match at any tier → {found:false}. `definition` is the
  first non-empty paragraph (not a missing frontmatter field).
- create_draft target path exists → `-2`/`-3` suffix; all suffixes exhausted →
  `WRITE_CONFLICT`.
- check: orphaned worktree older than 7 days → warning (does NOT fail CI);
  schema/dup/broken-link violation → non-zero exit (fails CI).
- export: empty repo → minimal `agents.md` with empty per-type sections;
  overwrites existing `./agents.md`.
- init idempotent: re-run over an existing `.project-knowledge/` doesn't clobber
  user docs (only adds missing templates/dirs).
- Server lifecycle: SIGTERM/SIGINT/stdin-close mid-create_draft → drain mutex,
  rollback the half-built worktree, exit. No orphaned worktree.

## Critical Paths
- Security invariant: no read/write escapes `.project-knowledge/` for any
  slug input. (A1 — must have a dedicated adversarial test suite.)
- create_draft never mutates the user's working tree or HEAD (worktree-based,
  not checkout-based). Regression-critical: a future refactor must not regress
  to `git checkout -b`.
- A/B kill-switch reproducibility: any contributor can run the harness and get
  the same token/hallucination numbers on the same repo+questions.
