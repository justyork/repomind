## Affected Pages/Routes

- `http://127.0.0.1:3847/` — workspace reader (unchanged PR1)
- `http://127.0.0.1:3847/` — draft editor after Edit / new draft (TipTap canvas)
- `POST /api/drafts/:id/publish` — publish-now flow without mandatory modal

## Key Interactions to Verify

- Open published page → **Edit** → single WYSIWYG canvas (no Markdown/Preview split)
- Type bold/heading/list → formatting visible inline
- Insert `[[` → wikilink autocomplete popover → select slug → rendered link chip
- Toolbar **Image** → upload → image appears in canvas
- **Saved** status after pause typing (~1s)
- **Publish** (primary) → page published without diff modal → reader shows new content
- **Publish ▾ → Review changes** → diff modal still works
- **View markdown** menu → shows serialized markdown matching disk after publish
- `e` keyboard shortcut still opens editor from reader

## Edge Cases

- Publish while "Saving…" visible → completes without data loss
- Autosave API error → toast, content retained in editor
- Doc with mermaid fence → round-trips as code block (no preview in editor PR1)
- Task list `- [ ]` / `- [x]` → matches reader rendering after publish
- Broken `related` slug on publish → inline error (existing validation)
- yaml/json page → Edit button hidden (unchanged)

## Critical Paths

1. **Edit → autosave → publish now → MCP `get_doc` body matches serialized markdown**
2. **Round-trip:** load fixture md → parse → serialize → byte-stable (± whitespace normalization)
3. **Lazy load:** initial page load bundle does not include TipTap; Edit triggers chunk fetch
4. **Dogfood:** edit one real SFC doc, publish, `repo-mind check` green
