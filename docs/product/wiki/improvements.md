# Improvements backlog

Friction and ideas from dogfood. **Do not** track release phases here — see [roadmap](roadmap.md).

## How to log

1. Reproduce in skyforge (or repo-mind `docs/`).
2. Add a row below: symptom, severity, workaround if any.
3. Promote to `technical/specs/` when scoped for implementation.

## Template

```markdown
### YYYY-MM-DD — Short title

- **Severity:** P0 | P1 | P2
- **Area:** tree | reader | editor | mcp | check | other
- **Symptom:** what happened
- **Workaround:** optional
- **Issue/PR:** optional link
```
