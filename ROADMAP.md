## AI и поиск

### AI-помощник по документации

- **Зачем:** задавать вопросы на естественном языке и получать ответы из `docs/` с цитатами и ссылками на страницы.
- **Заметки:**
  - Встроенный чат в Web UI (боковая панель или отдельная вкладка).
  - Ответы только из опубликованных файлов — те же границы, что у MCP (`search_docs` → `get_doc`).
  - Обязательные ссылки на источники (slug + заголовок); без цитаты — «не нашёл в документации».
  - Опционально: `repo-mind ask "…"` в CLI для терминала.
  - Открытые вопросы: свой API-ключ vs BYOK; локальная модель vs облако; кэш эмбеддингов.

### Семантический поиск

- **Зачем:** находить релевантные страницы по смыслу, а не только по ключевым словам.
- **Заметки:** индекс эмбеддингов поверх `docs/`; пересборка при publish; гибрид с текущим keyword-search.

### Умные подсказки при редактировании

- **Зачем:** при написании страницы предлагать связанные документы, wikilink'и и термины из глоссария.
- **Заметки:** на базе `explore_graph`, backlinks и (опционально) эмбеддингов; не автоправить текст без явного действия автора.

## UX и редактор

### История версий страницы в UI

- **Зачем:** смотреть diff и откатываться к прошлым версиям без ухода в `git log`.
- **Заметки:** git — источник истины; UI только обёртка над `git blame` / `git diff` для текущего файла.

### Комментарии и обсуждения на странице

- **Зачем:** обсуждать фрагменты документации без Confluence-style inline comments (пока нет).
- **Заметки:** хранить в `docs/` (frontmatter/block-annotation) или отдельно; не ломать MCP-read путь.

### Индикатор «устаревшей» страницы

- **Зачем:** видеть, что страница давно не обновлялась или ссылается на удалённые/deprecated сущности.
- **Заметки:** `updated` в frontmatter + link health; бейдж в дереве и на странице.

### Улучшения knowledge graph

- **Зачем:** быстрее ориентироваться в связях между документами.
- **Заметки:** фильтры по domain/type/tag; клик → открыть страницу; mini-graph на странице (локальное окружение узла).

### Глобальный поиск в UI

- **Зачем:** `Cmd+K` / `/` — мгновенный поиск по заголовкам, телу и метаданным с превью.
- **Заметки:** частично пересекается с keyboard nav (E2); можно объединить в command palette.

## Интеграции и экосистема

### Импорт из Confluence / Notion

- **Зачем:** миграция существующих баз знаний в `docs/`.
- **Заметки:** сейчас вне scope v4 ([roadmap](docs/product/wiki/roadmap.md)); отдельный одноразовый CLI `repo-mind import`.

### Webhooks и уведомления

- **Зачем:** Slack/Telegram/email при publish или при падении `repo-mind check`.
- **Заметки:** опционально; для команд, которые не сидят в git notifications.

### Плагины / расширения

- **Зачем:** кастомные блоки редактора, валидаторы frontmatter, доменные шаблоны без форка RepoMind.
- **Заметки:** долгий горизонт; начать с hook points в `check` и editor extensions.

## Операции и команда

### Роли и права (multi-user)

- **Зачем:** разграничить read / draft / publish для разных людей.
- **Заметки:** сейчас только `REPOMIND_UI_PASSWORD`; полноценный multi-user — отдельный эпик (SSO, audit log).

### Аналитика использования документации

- **Зачем:** понимать, какие страницы читают люди и какие запрашивают агенты.
- **Заметки:** локальный opt-in лог; без отправки контента наружу.

### Scheduled review / doc hygiene

- **Зачем:** напоминания «раз в квартал пересмотреть ADR-xxx»; отчёт по битым ссылкам и пустым README.
- **Заметки:** `repo-mind check --report` + CI cron; опционально issue в GitHub.

## Инфраструктура

### PWA / офлайн-чтение

- **Зачем:** читать опубликованные страницы без сети (dogfood в поездках).
- **Заметки:** service worker + кэш статики и API read-only.

### Упаковка «один бинарник»

- **Зачем:** `repo-mind ui` без отдельного Node/npm для нетехнических авторов.
- **Заметки:** pkg / deno compile / embedded assets.

- Batch-операции в дереве (переместить несколько, сменить domain)
- Полнотекстовый поиск по вложениям (PDF в `docs/`)
